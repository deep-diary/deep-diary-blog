Static visit

原来无法访问时候静态文件和媒体资源的配置

urlpatterns += static(settings.STATIC_URL, document_root=settings.STATICFILES_DIRS)
urlpatterns += static(settings.MEDIA_URL, document_root=settings.MEDIA_ROOT)

更改配置后，即可以访问

# 项目主路由 url.py
from django.views import static
from django.conf import settings


# 路由静态文件和图片debug 模式关闭的时候，需要添加如下2句话，才能正常访问静态目录和媒体目录
urlpatterns += [
    re_path('static/(?P<path>.*)$', serve, {'document_root': settings.STATIC_ROOT}, name='static'),
    re_path('media/(?P<path>.*)$', serve, {'document_root': settings.MEDIA_ROOT}, name='media')
]

Customize the configuration file

对于有些个人信息，比如一些api 的key，我们一般会单独设置一个配置文件，然后上传到github的时候，把这个配置文件给忽略掉

配置文件格式

比较简单的一种格式就是直接用python对象，这里的定义的位置如下图

使用配置文件

在项目中使用配置文件，只要从自定义配置文件中导入对象即可

from deep_diary.config import mysql_account

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.mysql',
        'NAME': mysql_account['DatabaseName'], 	 	# 这里使用配置文件
        'USER': mysql_account['USER'],   			# 这里使用配置文件
        'PASSWORD': mysql_account['PASSWORD'], 		# 这里使用配置文件
        'HOST': 'localhost',
        'PORT': '3306',
    },
}
