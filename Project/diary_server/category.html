在Django中，get_or_create()函数用于根据给定的查询参数获取对象，如果对象不存在，则创建一个新对象。查询参数和创建参数可以是不同的，但是需要满足模型字段的验证规则。

get_or_create()函数的使用方式如下：

pythonCopy code
obj, created = ModelName.objects.get_or_create(**lookup_params, defaults=creation_params)
1
2
  • ModelName是您的模型类的名称。
  • lookup_params是查询参数，用于查找现有对象。它是一个字典，包含字段名和对应的值。
  • creation_params是创建参数，用于在对象不存在时创建新对象。它是一个字典，包含字段名和对应的值。
  • obj是查询到的对象或新创建的对象。
  • created是一个布尔值，指示对象是否是新创建的。

注意事项：

  • 查询参数和创建参数可以是不同的，但是需要确保创建参数的值是有效的，并且满足模型字段的验证规则。
  • 如果对象已经存在，则get_or_create()函数将返回查询到的对象，并且created将为False。
  • 如果对象不存在，则根据创建参数创建一个新对象，并将created设置为True。
  • 创建参数defaults只有在对象不存在时才会生效，如果对象已经存在，则不会使用创建参数。
  • get_or_create()函数根据查询参数在数据库中执行查询操作，因此可能会产生数据库访问开销。

示例使用：

pythonCopy code# 查询参数
lookup_params = {'name': 'John'}

# 创建参数
creation_params = {'age': 25, 'email': 'john@example.com'}

# 查询现有对象或创建新对象
obj, created = MyModel.objects.get_or_create(**lookup_params, defaults=creation_params)

# 打印结果
print(obj)  # 查询到的对象或新创建的对象
print(created)  # 是否是新创建的对象

    # ----------------------add the single image to category----------------------
    def add_date_to_category(self, instance=None):
        # 1. get the instance and check is it OK to add to category
        instance, process = self.__is_OK_add_to_category__(instance=instance, field='dates')
        if not process:
            return
        # 2. get the date
        dates = instance.dates
        # 3. check whether the dates.year is existed or not, if not, create it
        year_obj, created = Category.objects.get_or_create(name=dates.year)
        # 4. deal with something if this is the first time to create the year
        if created:
            dict_year = {
                'parent': None,
                'level': 0,
                'is_leaf': False,
                'is_root': True,
                'owner': instance.user,
                'avatar': instance.src,
                'description': f'this is year category: {dates.year}',
            }
            year_obj.__dict__.update(dict_year)
            year_obj.save()
        # 5. add the instance to the year
        year_obj.imgs.add(instance)

        # 6. check whether the dates.month is existed or not, if not, create it
        month_obj, created = Category.objects.get_or_create(name=f'{dates.year}-{dates.month}', parent=year_obj)
        # 7. deal with something if this is the first time to create the month
        if created:
            dict_month = {
                # 'parent': Category.objects.get(name=dates.year),
                'level': 1,
                'is_leaf': False,
                'is_root': False,
                'owner': instance.user,
                'avatar': instance.src,
                'description': f'this is month category: {dates.month}',
            }
            month_obj.__dict__.update(dict_month)
            month_obj.save()
        # 8. add the instance to the month
        month_obj.imgs.add(instance)

        # 9. check whether the dates.day is existed or not, if not, create it
        day_obj, created = Category.objects.get_or_create(name=f'{dates.year}-{dates.month}-{dates.day}',
                                                          parent=month_obj)
        # 10. deal with something if this is the first time to create the day
        if created:
            dict_day = {
                # 'parent': Category.objects.get(name=dates.month),
                'level': 2,
                'is_leaf': True,
                'is_root': False,
                'owner': instance.user,
                'avatar': instance.src,
                'description': f'this is day category: {dates.day}',
            }
            day_obj.__dict__.update(dict_day)
            day_obj.save()
        # 11. add the instance to the day
        day_obj.imgs.add(instance)

在视图集中定义不同的查询集

由于category是树形结构，有很多子数，如果按每个节点遍历子节点，可能会引起一些重复，比如

  • a
    • b
      • c
  • b
    • c
  • c

如上，一个树，会出现3条记录，而且有些记录并非从根节点开始的

因此，如果需要解决这个问题，可以通过改变查询集的方式来实现：

就是在list状态下，只过滤根节点属性

在其它视图下，不进行过滤

class CategoryViewSet(viewsets.ModelViewSet):

            return self.queryset

    @staticmethod
    def get_cate_children(name, level=0):
        if level <= 1:  # represent the first level
            rst = Category.objects.get(name=name).get_children().annotate(value=Count('imgs')).values('name',
                                                                                                      'value').distinct().order_by(
                '-value'),
        else:
            rst = Category.objects.get(name=name).get_descendants().filter(level=level).annotate(
                value=Count('imgs')).values('name',
                                            'value').distinct().order_by('-value'),

        return rst
